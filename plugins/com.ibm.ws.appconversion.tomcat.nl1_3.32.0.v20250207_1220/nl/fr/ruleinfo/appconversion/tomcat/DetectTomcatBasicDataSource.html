Ne pas utiliser la source de données de base d'Apache Tomcat

Cette règle détecte l'utilisation du org.apache.tomcat.dbcp.dbcp.BasicDataSource dans les fichiers Java . Cette classe implémente l'interface de source de données standard de l'industrie javax.sql.DataSource . La source de données Apache Tomcat fournit des méthodes propriétaires que votre application peut utiliser.

Si vous n'utilisez pas de méthodes propriétaires, vous pouvez modifier votre code pour l'utiliser javax.sql.DataSource plutôt que de se lancer dans la Version de Tomcat .

Si vous utilisez les méthodes propriétaires et voulez continuer à les utiliser dans l'application, vous pouvez migrer la ressource pour qu'elle utilise la source de données Apache Commons. Pour utiliser Apache Commons, téléchargez le fichier jar common-dbcp et configurez votre ressource en vue de l'utilisation de cette source de données. Dans votre code Java , modifiez les références de org.apache.tomcat.dbcp.dbcp.BasicDataSource à org.apache.commons.dbcp.BasicDataSource .

Vous pouvez aussi reconfigurer votre ressource pour qu'elle utilise des pools de connexions WebSphere afin d'améliorer les performances et de limiter les problèmes liés aux unités d'exécution et à l'évolutivité. Avec cette solution, des informations et des statistiques sur la source de données sont disponibles dans votre application via la table de définitions ConnectionFactory MBean.

Pour plus d'informations, voir :

  • Apache Commons - The DBCP Component
