Ne pas utiliser d'API ni de packages Apache Tomcat

Cette règle analyse les fichiers Java pour détecter l'utilisation des packages propriétaires Apache Tomcat suivants :

  • org.apache.tomcat
  • org.apache.catalina
  • org.apache.coyote
  • org.apache.el
  • org.apache.jasper
  • org.apache.jk
  • org.apache.juli
  • org.apache.naming

Si votre application utilise des classes dans ces packages, le code doit être modifié afin d'utiliser les API du kit de développement Java (JDK), les API de WebSphere Application Server ou les API à code source ouvert.

Evaluez le code qui utilise l'API propre à Tomcat et changez-le pour utiliser une API non propriétaire ou une API WebSphere Application Server.

Pour plus d'informations, voir :

  • Apache Tomcat 8.0 API (documentation Java).
  • Apache Tomcat 9.0 API (documentation Java).
