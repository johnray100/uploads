Ne pas utiliser la méthode doUpgrade d'Apache Tomcat WsServerContainer

Liberty version 8.5.5.7 et WebSphere Application Server traditionnel version 9.0 introduisent la méthode WebSocket com.ibm.websphere.wsoc.WsWsocServerContainer doUpgrade, laquelle permet à un servlet ou à un filtre de demander que la demande HTTP actuelle soit mise à niveau vers une session WebSocket. Apache Tomcat fournit une fonction similaire par le biais de la méthode org.apache.tomcat.websocket.server.WsServerContainer doUpgrade.

Cette règle marque la méthode Tomcat WsServerContainer doUpgrade afin que vous puissiez la migrer vers la méthode WsWsocServerContainer doUpgrade.

La méthode WebSphere Application Server est définie comme suit : 


com.ibm.websphere.wsoc.WsWsocServerContainer

public void doUpgrade(HttpServletDemande, HttpServletRéponse, ServerEndpointConfig sec, java.util.Map<java.lang.String, java.lang.String> pathParams) émet une exception ServletException, java.io.IOException }

Pour plus d'informations et un lien vers la documentation Java, voir
