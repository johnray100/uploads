Non utilizzare i package ed API Apache Tomcat

Questa regola esegue la scansione dei file Java per rilevare l'utilizzo dei seguenti package proprietari Apache Tomcat:

  • org.apache.tomcat
  • org.apache.catalina
  • org.apache.coyote
  • org.apache.el
  • org.apache.jasper
  • org.apache.jk
  • org.apache.juli
  • org.apache.naming

Quando l'applicazione utilizza le classi in questi package, il codice deve essere modificato per utilizzare le API fornite da JDK (Java development kit), le API fornite da WebSphere Application Server o le API open source.

Valutare il codice utilizzando l'API specifica Tomcat e modificarlo per utilizzare un'API non proprietaria o un'API WebSphere Application Server.

Per ulteriori informazioni, consultare:

  • Apache Tomcat 8.0 API Documentazione Java.
  • Apache Tomcat 9.0 API Documentazione Java
