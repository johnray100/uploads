Utilizza i descrittori di distribuzione Java EE per definire i listener del ciclo di vita di contesto

Questa regola contrassegna tutti Context <Listener> nel file META-INF/context.xml .

In Apache Tomcat, gli oggetti Java che devono sapere quando il loro contesto viene avviato o arrestato vengono dichiarati includendo un Listener nel contesto. Il nome classe specificato nel Listener L'elemento implementa il org.apache.catalina.LifecycleListener :NONE. A tale classe verrà notificata la ricorrenza dei corrispondenti eventi del ciclo di vita. Tenere presente che Non utilizzare la regola Apache Tomcat package e API contrassegna le ricorrenze dell'utilizzo di org.apache.catalina.LifecycleListener .

La funzionalità Java EE analoga è disponibile utilizzando javax.servlet.ServletContextListener e ServletContextEvent e il corrispondente Listener voci nel file web.xml .

Tomcat consente Context <Listener> elementi da definire in queste ubicazioni:

  • il file META-INF/context.xml
  • Il file di configurazione XML dell'applicazione nella directory Tomcat conf
  • Gli <Host> in conf/server.xml

Solo indicatori dello strumento di migrazione Listener nel file META-INF/context.xml . Listener Le informazioni definite nelle altre ubicazioni possono essere copiate nel file META-INF/context.xml per poterle elaborare.

L'azione consigliata consiste nel convertire i file Java utilizzando org.apache.catalina.LifecycleListener per utilizzare javax.servlet.ServletContextListener e per aggiungere le voci listener appropriate al file web.xml per l'applicazione.
