Establecer el ámbito de compartimiento en las referencias de recursos

Esta regla marca resource-ref elementos encontrados en el archivo WEB-INF/web.xml a los que les falta el res-sharing-scope . El servidor Apache Tomcat toma de forma predeterminada el ámbito de compartición de recursos Shareable . El ámbito de compartición de recursos se define como un atributo opcional en Java EE. Un arreglo rápido para esta regla está disponible en el explorador de orígenes. Puede utilizar el arreglo rápido para obtener el mismo ámbito compartible en WebSphere tradicional y Liberty que el que tenía en Tomcat.

El arreglo rápido añade el res-sharing-scope si todavía no se ha establecido y define la conexión como Shareable .

Este es un ejemplo de una referencia de recurso que se señalará:
< span class="Code"> < ref-recurso>
< span class= "indent4"> < /span> < descripción>
Base de datos de ejemplo</description>
<res-ref-name>jdbc/exampleDB</res-ref-name>
<res-type>javax.sql.DataSource</res-type>
<res-auth>Contenedor</res-auth>
</resource-ref>


El arreglo rápido añade el elemento res-sharing-scope:
< span class="Code"> < ref-recurso>
< span class= "indent4"> < /span> < descripción>
Base de datos de ejemplo</description>
<res-ref-name>jdbc/exampleDB</res-ref-name>
<res-type>javax.sql.DataSource</res-type>
<res-auth>Contenedor</res-auth>
<res-sharing-scope>Compartible</res-sharing-scope>
</resource-ref>

Para obtener más información sobre cómo compartir conexiones de recursos en WebSphere Application Server, consulte Conexiones compartibles y no compartibles.
