No utilizar el método doUpgrade de WsServerContainer de Apache Tomcat

Liberty V8.5.5.7 y WebSphere Application Server V9.0 tradicional introducen el método com.ibm.websphere.wsoc.WsWsocServerContainer doUpgrade de WebSocket, que permite a un servlet o filtro solicitar que la solicitud HTTP actual se actualice a una sesión de WebSocket. Apache Tomcat proporciona capacidad similar mediante el método org.apache.tomcat.websocket.server.WsServerContainer doUpgrade.

Esta regla marca el método Tomcat WsServerContainer doUpgrade para que pueda migrarlo al método WsWsocServerContainer doUpgrade.

El método de WebSphere Application Server se define de la manera siguiente: 


com.ibm.websphere.wsoc.WsWsocServerContainer

public void doUpgrade(HttpServletRequest request, HttpServletResponse response, ServerEndpointConfig sec, java.util.Map<java.lang.String, java.lang.String> pathParams) throws ServletException, java.io.IOException }
