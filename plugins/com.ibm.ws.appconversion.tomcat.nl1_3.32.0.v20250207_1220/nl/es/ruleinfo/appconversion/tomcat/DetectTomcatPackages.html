No utilizar API y paquetes de Apache Tomcat

Esta regla explora los archivos Java para detectar el uso de los siguientes paquetes de propietario de Apache Tomcat:

  • org.apache.tomcat
  • org.apache.catalina
  • org.apache.coyote
  • org.apache.el
  • org.apache.jasper
  • org.apache.jk
  • org.apache.juli
  • org.apache.naming

Cuando su aplicación utiliza clases contenidas en estos paquetes, se debe cambiar el código para utilizar las API proporcionadas por el Kit de desarrollo de Java (JDK), las API proporcionadas por WebSphere Application Server o las API de código abierto.

Evalúe el código utilizando la API específica de Tomcat y cámbielo para utilizar una API no exclusiva o una API de WebSphere Application Server.

Para obtener información adicional, consulte:

  • API de Apache Tomcat 8.0.
  • API de Apache Tomcat 9.0.
