Validar o resultado de concatenação com getRealPath("")

Essa regra sinaliza o código Java que tem referências para os métodos e os parâmetros a seguir:

  • javax.servlet.http.HttpServletRequest.getRealPath ("")
  • javax.servlet.ServletContext.getRealPath ("")
  • javax.servlet.ServletRequest.getRealPath ("")

No Tomcat, essas chamadas de método retornarão uma Sequência que termina com uma barra. No Liberty, o valor de retorno não terminará com uma barra. Como resultado, a concatenação de Sequências com esse método pode apontar para um arquivo válido no Tomcat, mas não no Liberty. Para cada chamada de método sinalizada, verifique se qualquer código está usando as contas de resultado para a ausência de uma barra à direita no Liberty.

Por exemplo, suponha que você tenha instalado um war explodido em /path/app.war. Em seguida, chamar getRealPath("") apresentará os seguintes resultados:

  • Tomcat: /path/app.war/
  • Liberty: /path/app.war
