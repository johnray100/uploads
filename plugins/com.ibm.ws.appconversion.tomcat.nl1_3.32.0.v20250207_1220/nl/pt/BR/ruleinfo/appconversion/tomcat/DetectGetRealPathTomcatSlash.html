Validar o resultado de concatenação com getRealPath("/")

Essa regra sinaliza o código Java que tem referências para os métodos e os parâmetros a seguir:

  • javax.servlet.http.HttpServletRequest.getRealPath ("/")
  • javax.servlet.ServletContext.getRealPath ("/")
  • javax.servlet.ServletRequest.getRealPath ("/")

No Tomcat, essas chamadas de método retornarão uma Sequência que termina com uma barra. No Liberty e no WebSphere tradicional, o valor de retorno não terminará com uma barra. Como resultado, a concatenação de Sequências com esse método pode apontar para um arquivo válido no Tomcat, mas não no WebSphere. Para cada chamada de método sinalizada, verifique se qualquer código usando as contas de resultado para a falta de barra à direita no WebSphere.

Por exemplo, suponha que você tenha instalado um war explodido em /path/app.war. Em seguida, chamar getRealPath("/") apresentará os seguintes resultados:

  • Tomcat: /path/app.war/
  • WebSphere: /path/app.war
