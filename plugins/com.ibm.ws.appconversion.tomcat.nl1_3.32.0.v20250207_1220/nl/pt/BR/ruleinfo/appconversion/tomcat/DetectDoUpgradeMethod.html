Não usar o método Apache Tomcat WsServerContainer doUpgrade

O Liberty V8.5.5.7 e o WebSphere Application Server V9.0 tradicional apresentam o método WebSocket com.ibm.websphere.wsoc.WsWsocServerContainer doUpgrade, que possibilita que o servlet ou o filtro solicite o upgrade da solicitação de HTTP atual para uma sessão de WebSocket. O Apache Tomcat fornece capacidade semelhante por meio do método org.apache.tomcat.websocket.server.WsServerContainer doUpgrade.

Essa regra sinaliza o método WsServerContainer doUpgrade do Tomcat para que seja possível migrá-lo para o método WsWsocServerContainer doUpgrade.

O método do WebSphere Application Server é definido como segue: 


com.ibm.websphere.wsoc.WsWsocServerContainer

public void doUpgrade(HttpServletRequest request, HttpServletResponse response, ServerEndpointConfig sec, java.util.Map<java.lang.String, java.lang.String> pathParams) lança ServletException, java.io.IOException }
