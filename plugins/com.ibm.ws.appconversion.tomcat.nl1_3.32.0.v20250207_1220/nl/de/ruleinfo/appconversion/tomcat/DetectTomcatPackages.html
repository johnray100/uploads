Apache-Tomcat-Pakete und -APIs nicht verwenden

Diese Regel durchsucht Java-Dateien, um festzustellen, ob folgenden proprietären Apache-Tomcat-Pakete verwendet werden:

  • org.apache.tomcat
  • org.apache.catalina
  • org.apache.coyote
  • org.apache.el
  • org.apache.jasper
  • org.apache.jk
  • org.apache.juli
  • org.apache.naming

Wenn Ihre Anwendung Klassen aus diesen Paketen verwendet, muss der Code so geändert werden, dass von Java Development Kit (JDK) bereitgestellte APIs, von WebSphere Application Server bereitgestellte APIs oder Open-Source-APIs verwendet werden.

Werten Sie den Code, der die Tomcat-spezifische API verwendet, aus, und ändern Sie ihn so, dass er eine nicht proprietäre API oder eine API von WebSphere Application Server verwendet.

Weitere Informationen finden Sie in den folgenden Dokumenten:

  • Java-Dokumentation - Apache Tomcat 9.0 API
