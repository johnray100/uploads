Bereich für gemeinsame Nutzung in Ressourcenreferenzen festlegen

Diese Regel markiert Folgendes: resource-ref Elemente in der Datei WEB-INF/web.xml , in denen die res-sharing-scope Element. Apache Tomcat Server setzt den Geltungsbereich für die gemeinsame Nutzung von Ressourcen standardmäßig auf Shareable . Der Bereich für gemeinsame Ressourcennutzung ist als optionales Attribut in Java EEdefiniert. Es ist eine Schnellkorrektur für diese Regel im Quellenscanner verfügbar. Sie können die Schnellkorrektur verwenden, um in WebSphere Traditional und Liberty denselben gemeinsam nutzbaren Bereich wie in Tomcat abzurufen.

Die Schnellkorrektur fügt die res-sharing-scope Element, wenn es noch nicht festgelegt ist, und definiert die Verbindung als Shareable .

Im Folgenden sehen Sie ein Beispiel für eine Ressourcenreferenz, die markiert wird:
< span class="Code"> < Ressourcenref>
< span class= "indent4"> < /span> < description>
Beispieldatenbank</description>
<res-ref-name>jdbc/exampleDB</res-ref-name>
<res-type>javax.sql.DataSource</res-type>
<res-auth>Container</res-auth>
</resource-ref>


Die Schnellkorrektur fügt das Element "res-sharing-scope" hinzu:
< span class="Code"> < Ressourcenref>
< span class= "indent4"> < /span> < description>
Beispieldatenbank</description>
<res-ref-name>jdbc/exampleDB</res-ref-name>
<res-type>javax.sql.DataSource</res-type>
<res-auth>Container</res-auth>
<res-sharing-scope>Gemeinsam nutzbar</res-sharing-scope>
</resource-ref>

Weitere Informationen zur gemeinsamen Nutzung von Ressourcenverbindungen in WebSphere Application Server finden Sie im Artikel Nicht gemeinsam und gemeinsam nutzbare Verbindungen.
