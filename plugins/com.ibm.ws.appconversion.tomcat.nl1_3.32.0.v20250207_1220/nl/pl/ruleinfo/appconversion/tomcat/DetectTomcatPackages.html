Nie używaj pakietów i interfejsów API produktu Apache Tomcat

Ta reguła skanuje pliki Java, aby wykryć użycie następujących prawnie zastrzeżonych pakietów produktu Apache Tomcat:

  • org.apache.tomcat
  • org.apache.catalina
  • org.apache.coyote
  • org.apache.el
  • org.apache.jasper
  • org.apache.jk
  • org.apache.juli
  • org.apache.naming

Jeśli aplikacja używa klas w tych pakietach, należy zmienić kod w taki sposób, aby używał interfejsów API z pakietu Java Development Kit (JDK), interfejsów API udostępnianych przez serwer WebSphere Application Server lub interfejsów API Open Source.

Należy ocenić kod korzystający z interfejsu API specyficznego dla produktu Tomcat i zmienić go w taki sposób, aby użyć niezastrzeżonego prawnie interfejsu API lub interfejsu API serwera WebSphere Application Server.

Dodatkowe informacje na ten temat:

  • Interfejs API Apache Tomcat 8.0, dokumentacja języka Java.
  • Interfejs API Apache Tomcat 9.0, dokumentacja języka Java.
