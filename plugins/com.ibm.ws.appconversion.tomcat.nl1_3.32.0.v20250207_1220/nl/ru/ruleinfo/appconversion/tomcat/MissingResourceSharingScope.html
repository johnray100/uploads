Задать область совместного доступа для ссылок на ресурсы

Это правило помечает resource-ref Элементы, найденные в файле WEB-INF/web.xml , отсутствуют res-sharing-scope . СерверApache Tomcat по умолчанию использует область совместного использования ресурсов Shareable . Область совместного использования ресурсов определена как необязательный атрибут в Java EE. В сканере исходного кода доступно быстрое исправление для этого правила. Для получения в WebSphere traditional и Liberty области совместного использования, аналогичной области в Tomcat, можно использовать быстрое исправление.

Быстрое исправление добавляет res-sharing-scope элемент, если он еще не задан и определяет соединение как Shareable .

Пример помечаемой ссылки на ресурс:
< span class="Code"> < resource-ref>
< span class= "indent4"> < /span> < description>
Пример базы данных</description>
<res-ref-name>jdbc/exampleDB</res-ref-name>
<res-type>javax.sql.DataSource</res-type>
<res-auth>Контейнер</res-auth>
</resource-ref>


Процедура быстрого исправления добавляет элемент res-sharing-scope:
< span class="Code"> < resource-ref>
< span class= "indent4"> < /span> < description>
Пример базы данных</description>
<res-ref-name>jdbc/exampleDB</res-ref-name>
<res-type>javax.sql.DataSource</res-type>
<res-auth>Контейнер</res-auth>
<res-sharing-scope>Общий доступ</res-sharing-scope>
</resource-ref>

Дополнительная информация о предоставлении общего доступа к соединениям с ресурсами в WebSphere Application Server приведена в разделе Соединения с совместным доступом и без совместного доступа.
