Setaţi partajarea domeniului pe referinţele de resurse

Această regulă semnalează resource-ref elemente găsite în fișierul WEB-INF/web.xml care lipsesc res-sharing-scope element. Apache Tomcat server defaulturi de partajare a resurselor domeniul de aplicare la Shareable . Domeniul de aplicare al partajării resurselor este definit ca un atribut opțional în Java EE. O corecţie rapidă pentru această regulă este disponibilă în scanerul de sursă. Puteţi folosi corecţia rapidă pentru a obţine în WebSphere tradiţional şi Liberty domeniul partajabil pe care îl aveaţi în Tomcat.

Remedierea rapidă adaugă res-sharing-scope element în cazul în care nu este deja stabilit și definește conexiunea ca Shareable .

Acesta este un exemplu de referinţă de resurse care va fi semnalat:
< span class="Code"> < resource-ref>
< span class= "indent4" > < /span> < description>
Exemplu Baza de date</description>
<res-ref-name>jdbc/exampleDB</res-ref-name>
<res-type>javax.sql.DataSource</res-type>
<res-auth>Container</res-auth>
</resource-ref>


Corecţia rapidă adaugă elementul res-sharing-scope:
< span class="Code"> < resource-ref>
< span class= "indent4" > < /span> < description>
Exemplu Baza de date</description>
<res-ref-name>jdbc/exampleDB</res-ref-name>
<res-type>javax.sql.DataSource</res-type>
<res-auth>Container</res-auth>
<res-sharing-scope>Shareable</res-sharing-scope>
</resource-ref>
