Utilizaţi descriptori de implementare Java EE pentru a defini ascultători de cicluri de viaţă de context

Această regulă steaguri toate Context <Listener> elemente în META-INF/context.xml .

In Apache Tomcat, Java objects that need to know when their context is started or stopped are declared by including a Listener element în cadrul contextului. Denumirea clasei specificată în Listener element implementează org.apache.catalina.LifecycleListener Interfață. Această clasă va fi notificată asupra apariţiei evenimentelor ciclului de viaţă corespunzător. Rețineți că Nu utilizați Apache Tomcat și APIs steaguri de reguli de utilizare a utilizării de org.apache.catalina.LifecycleListener .

Funcționalitatea analogous Java EE utilizând javax.servlet.ServletContextListener și ServletContextEvent și corespondentul Listener intrări în fișierul web.xml .

Tomcat permite Context <Listener> elemente care trebuie definite în aceste locații:

  • Fişierul META-INF/context.xml
  • Fişierul de configuraţie XML al aplicaţiei din directorul Tomcat conf
  • The <Host> element în conf/server.xml

Instrumentul de migrare numai steaguri Listener elemente în META-INF/context.xml . Listener informații definite în celelalte locații pot fi copiate în fișierul META-INF/context.xml pentru ca acesta să fie procesat.

Acțiunea recomandată este de a converti fișierele Java folosind org.apache.catalina.LifecycleListener pentru utilizare javax.servlet.ServletContextListener și pentru a adăuga intrările corespunzătoare ascultătorului la fișierul web.xml pentru aplicație.
