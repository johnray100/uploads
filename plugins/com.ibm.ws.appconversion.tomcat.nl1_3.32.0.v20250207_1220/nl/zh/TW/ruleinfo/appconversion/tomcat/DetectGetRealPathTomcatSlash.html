使用 getRealPath("/") 驗證連結的結果

此規則會標示參照下列方法及參數的 Java 程式碼：

  • javax.servlet.http.HttpServletRequest.getRealPath("/")
  • javax.servlet.ServletContext.getRealPath("/")
  • javax.servlet.ServletRequest.getRealPath("/")

在 Tomcat 中，這些方法呼叫會傳回以正斜線結尾的字串。在 Liberty 及 WebSphere Traditional 中，回覆值將不會以正斜線結尾。因此，使用此方法的字串連結在 Tomcat 中可能會指向有效檔案，但在 WebSphere 中不會指向有效檔案。對於標示的每個方法呼叫，請檢查使用該結果的任何程式碼在 WebSphere 中皆會對缺少尾端斜線負責。

例如，假設您將分解的 war 已安裝至 /path/app.war。則呼叫 getRealPath("/") 將具有下列結果：

  • Tomcat：/path/app.war/
  • WebSphere：/path/app.war
