Java EE 배치 디스크립터를 사용하여 컨텍스트 라이프사이클 리스너 정의

이 규칙은 모든 Context <Listener> 요소를 META-INF/context.xml 파일에 추가하십시오.

Apache Tomcat에서 컨텍스트가 시작되거나 중지되는 시기를 알아야 하는 Java 오브젝트는 다음을 포함하여 선언됩니다. Listener 요소를 사용할 수 있습니다. 다음에 지정된 클래스 이름 Listener 요소는 org.apache.catalina.LifecycleListener 인터페이스를 구현한 항목입니다. 이 클래스는 해당 라이프사이클 이벤트의 발생에 관한 알림을 받게 됩니다. 다음에 유의하십시오. Apache Tomcat 패키지 및 API 규칙을 사용하지 마십시오. org.apache.catalina.LifecycleListener .

다음을 사용하여 유사한 Java EE 기능을 사용할 수 있습니다. javax.servlet.ServletContextListener 및 ServletContextEvent 및 해당하는 Listener 항목을 web.xml 파일에 추가하십시오.

Tomcat 은 Context <Listener> 다음 위치에 정의될 요소:

  • META-INF/context.xml 파일
  • Tomcat conf 디렉토리의 애플리케이션 XML 구성 파일
  • 다음 <Host> conf/server.xml 의 요소

마이그레이션 도구 전용 플래그 Listener 요소를 META-INF/context.xml 파일에 추가하십시오. Listener 다른 위치에 정의된 정보를 처리하기 위해 META-INF/context.xml 파일에 복사할 수 있습니다.

권장 조치는 다음을 사용하여 Java 파일을 변환하는 것입니다. org.apache.catalina.LifecycleListener 사용 javax.servlet.ServletContextListener 애플리케이션의 web.xml 파일에 적절한 리스너 항목을 추가합니다.
