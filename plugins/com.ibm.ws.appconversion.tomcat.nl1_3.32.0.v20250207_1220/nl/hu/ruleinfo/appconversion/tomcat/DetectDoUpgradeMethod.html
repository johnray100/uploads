Ne használja az Apache Tomcat WsServerContainer doUpgrade módszerét

A Liberty V8.5.5.7 és a hagyományos WebSphere Application Server V9.0 kiadásokban került bevezetésre a com.ibm.websphere.wsoc.WsWsocServerContainer doUpgrade WebSocket módszer, ami lehetővé teszi a szervlet vagy szűrő számára, hogy kérje az aktuális HTTP kérés frissítését WebSocket munkamenetre. Az Apache Tomcat a org.apache.tomcat.websocket.server.WsServerContainer doUpgrade módszeren keresztül ad hasonló képességet.

Ez a szabály megjelöli a Tomcat WsServerContainer doUpgrade módszerét, hogy átállíthassa azt a WsWsocServerContainer doUpgrade módszerre.

A WebSphere Application Server módszer meghatározása a következő: 


com.ibm.websphere.wsoc.WsWsocServerContainer

Nyilvános hézag doUpgrade(HttpServletRequest request, 
                      HttpServletResponse response,
                      ServerEndpointConfig sec, 
                      java.util.Map<java.lang.String,
                      java.lang.String> pathParams) 
                        Dob ServletException, java.io.IOException

}

További információkat és a Java dokumentáció hivatkozását a következő helyen találja:

  • WebSocket alkalmazások fejlesztése Liberty rendszerben.
  • WebSocket alkalmazások fejlesztése hagyományos WebSphere kiszolgálón.
