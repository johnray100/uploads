Použít deskriptory implementace Java EE k definování modulů listener životního cyklu kontextu

Toto pravidlo značí vše Context <Listener> prvky v souboru META-INF/context.xml .

Na serveru Apache Tomcatjsou objekty Java , které potřebují vědět, kdy je jejich kontext spuštěn nebo zastaven, deklarovány zahrnutím Listener prvek v rámci kontextu. Název třídy uvedený v Listener prvek implementuje org.apache.catalina.LifecycleListener rozhraní. Této třídě bude oznamován výskyt odpovídajících událostí životního cyklu. Všimněte si, že Nepoužívat pravidlo Apache Tomcat balíky a rozhraní API značí výskyty použití org.apache.catalina.LifecycleListener .

Analogická funkčnost prostředí Java EE je k dispozici pomocí javax.servlet.ServletContextListener a ServletContextEvent a odpovídající Listener položky v souboru web.xml .

Tomcat umožňuje Context <Listener> prvky, které mají být definovány v těchto umístěních:

  • Soubor META-INF/context.xml
  • Konfigurační soubor XML aplikace v adresáři Tomcat conf
  • V případě <Host> prvek v souboru conf/server.xml

Příznaky pouze nástroje migrace Listener prvky v souboru META-INF/context.xml . Listener informace definované v jiných umístěních lze zkopírovat do souboru META-INF/context.xml , aby bylo možné je zpracovat.

Doporučenou akcí je převod souborů Java pomocí org.apache.catalina.LifecycleListener k použití javax.servlet.ServletContextListener a chcete-li přidat odpovídající položky modulu listener do souboru web.xml pro aplikaci.
